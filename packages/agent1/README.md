# Agent 1

Agent 1 extracts formal requirements from a natural language conversation between a human user and the vibecoding agent, and translates them into a JSON-structured list of testing steps.

## Usage

```bash
uv run agent1 --model models/gemini-3-flash-preview --logging --no-cache
```

### Arguments

| Argument     | Required | Default                         | Description                                  |
| ------------ | -------- | ------------------------------- | -------------------------------------------- |
| `--model`    | ❌        | `models/gemini-3-flash-preview` | LLM model to use                             |
| `--logging`  | ❌        | `False`                         | Enable logging (also via `LOGGING=true` env) |
| `--no-cache` | ❌        | `False`                         | Disable DSPy caching for fresh LLM responses |

### Environment Variables

| Variable  | Description                              |
| --------- | ---------------------------------------- |
| `LOGGING` | Set to `true` to enable logging globally |

## Input

- A natural language conversation between a human user and the vibecoding agent
- The URL of the vibecoded app to test

## Output

A JSON-structured list of steps agent 2 must execute in order to test the vibecoded app

## File Structure

```
packages/agent1/src/agent1/
├── agent1.py                      # Main inference logic (loads compiled model)
├── agent1_prompt.py               # Training script (compiles with DSPy)
├── qa_architect_compiled.json     # Optimized model state (few-shot demos)
└── models.py                      # Pydantic schemas
```

### Key Files

- **`agent1_prompt.py`**: Contains training examples, validation metric, and DSPy module definition. Run with `agent1-compile` to optimize.
- **`qa_architect_compiled.json`**: Generated by DSPy's `BootstrapFewShot` optimizer. Contains the selected few-shot demonstrations and model configuration.
- **`agent1.py`**: Production inference code. Loads `qa_architect_compiled.json` and uses it for fast inference.

## Prompt modification

We use DSPy to optimise Agent 1's prompt with few-shot examples. To modify the prompt:

1. Edit `agent1_prompt.py` to add/remove training examples or change the prompt structure.
2. Recompile the model by running:

   ```bash
   uv run agent1-compile
   ```

The compiled prompt is now saved in `qa_architect_compiled.json` and will be used for subsequent agent runs.

See [DSPy documentation](https://dspy-docs.vercel.app/) for more optimization strategies.
